# BIZ AI Assistant

## "PC 한 대 깔아드립니다" — 기업용 AI 비서 세팅 사업

> 직원이 재료만 던지면, AI가 완성된 문서를 만들어서 돌려주는 구조

---

## 한줄 요약

Windows PC 한 대에 AI를 세팅해서 기업에 납품.
엑셀, 한글, 워드, 녹음파일 — 뭘 줘도 AI가 알아서 분석하고 문서를 생성한다.

---

## 이 사업이 왜 되나?

### 시장 상황 (2026년)
- 국내 기업 85%가 생성형 AI 도입 원함
- 근데 "어떻게 해야할지 모름"이 가장 큰 장벽
- 대형 컨설팅(SK AX, 솔트룩스): 수천만원~억 단위 → 중소기업 접근 불가
- 교육업체(한국GPT협회 등): 가르치기만 함, 직접 세팅 안 해줌
- 정부 바우처: 신청 복잡, 6개월 기다림

### 빈틈
**"저렴하게 직접 깔아주는" 서비스가 없다.**

### 우리 포지션: "동네 IT 형"
- 150~200만원이면 끝 (대형 대비 1/10)
- 3~5일이면 완료 (대형은 수개월)
- 교육이 아니라 직접 세팅 + 운영
- 텔레그램 채팅으로 바로 쓸 수 있는 실용적 결과물

---

## 핵심 기능: "양식 + 데이터 → 새 문서 자동 생성"

### 이런 게 된다

**회의록 자동 작성**
```
직원이 텔레그램으로:
"회의록 작성해줘
 - 녹음파일: (음성파일 전송)
 - 장소: 본사 3층 회의실
 - 시간: 4/16 오후 2시
 - 참석자: 김과장, 이대리, 박부장"

→ AI가:
1. 음성 → 텍스트 변환 (STT)
2. 안건/결정사항/TODO 자동 분류
3. 기업 양식에 맞춰 회의록 작성
4. 한글(.hwp) 또는 워드(.docx) 파일 생성
5. 텔레그램으로 완성 파일 회신
```

**보고서 자동 생성**
```
공유폴더에 매출 엑셀 넣으면
→ AI가 읽고 분석
→ 월간 매출 보고서 자동 생성 (차트 포함)
→ 텔레그램으로 발송
```

**서식 학습**
```
처음에: "우리 회의록 양식 이거야" (파일 전달)
이후: 그 양식 그대로 새 파일 찍어냄
매번 같은 포맷으로 생성
```

### 생성 가능한 파일 형식
| 형식 | 지원 | 용도 |
|------|------|------|
| 엑셀 (.xlsx) | O | 표, 차트, 수식 포함 |
| 워드 (.docx) | O | 보고서, 회의록 |
| 한글 (.hwp) | O | 관공서/건설 서류 |
| PDF | O | 최종 배포용 |
| CSV/JSON | O | 데이터 내보내기 |
| 이미지/차트 | O | 시각화 결과물 |

### 분석 가능한 입력
| 입력 | 처리 |
|------|------|
| 엑셀/CSV 파일 | 데이터 분석, 이상치 탐지, 트렌드 |
| 음성 녹음 | STT → 텍스트 → 회의록 |
| 한글/워드 문서 | 내용 분석, 요약, 분류 |
| 이미지 | OCR → 텍스트 추출 |
| DB 데이터 | 자연어 쿼리 ("이번 달 매출 뽑아줘") |

## 제공 서비스

### 1. 데이터 분석
- Excel/CSV/JSON 파일 자동 분석
- 이상치 탐지, 패턴 분석, 트렌드 파악
- 시각화 차트 자동 생성
- 업종별 맞춤 분석 (매출/재고/인사/공정)

### 2. 문서 자동 생성
- 보고서 자동 작성 (일일/주간/월간)
- 회의록 (녹음 → 문서)
- 계약서/견적서 초안
- 기업 양식 학습 → 동일 포맷 반복 생성

### 3. 파일 분류 & 정리
- 공유폴더 파일 자동 분류
- 문서 내용 기반 태깅
- 중복 파일 탐지
- 폴더 구조 자동 정리

### 4. DB 연동 분석
- 기업 DB 직접 연결 (MySQL/PostgreSQL/MSSQL)
- 자연어로 쿼리 ("거래처별 미수금 뽑아줘")
- 정기 모니터링 & 이상 시 알림

### 5. 정기 자동화
- 매일 아침 일일 리포트 자동 생성
- 매주 월요일 주간 분석 보고
- 새 파일 감지 → 자동 처리
- 텔레그램으로 결과 발송

## 시스템 구조

```
Windows PC (기업 현장 — 기존 PC 활용 가능)
│
├── WSL2 (Ubuntu — Windows 안의 리눅스)
│   ├── Claude Code (AI 엔진)
│   ├── 텔레그램 봇 (직원↔AI 소통 채널)
│   ├── 크론잡 (자동 스케줄)
│   │   ├── 매일 09:00 — 일일 리포트
│   │   ├── 매주 월 — 주간 분석
│   │   └── 파일 감시 — 새 파일 자동 처리
│   └── 분석 도구 (Python, Node.js)
│
├── 공유 폴더 (사내 네트워크)
│   ├── /입고/ — 직원이 파일 넣는 곳
│   ├── /결과/ — AI가 처리 결과 저장
│   └── /보고서/ — 자동 생성 보고서
│
├── DB 연결 (선택)
│   └── 기존 ERP/회계 시스템 연동
│
└── 원격 관리 (RDP/SSH)
    └── 외부에서 유지보수
```

### 사용 흐름
```
직원 → 텔레그램 메시지 or 공유폴더에 파일 드롭
  ↓
AI가 분석/처리/문서 생성
  ↓
결과를 텔레그램으로 회신 or 공유폴더에 저장
```

## 업종별 활용 예시

### 건설업
- 일일 작업일보 자동 정리
- 안전서류(TBM, 위험성평가) 자동 생성 → 한글(.hwp)
- 자재 입출고 데이터 분석
- 공정률 자동 계산 & 보고

### 제조업
- 생산 데이터 이상치 탐지 → 즉시 알림
- 불량률 추이 분석 & 시각화
- 재고 현황 리포트 자동 생성
- 거래처별 매출/납기 분석

### 도매/유통
- 매출/매입 자동 대조
- 거래처별 미수금 알림
- 상품별 판매 추이 분석
- 재고 부족 예측 & 발주 알림

### 사무/서비스
- 회의록 자동 작성 (녹음 → 문서)
- 이메일/문서 자동 분류
- 고객 문의 분류 & 통계
- 직원 근태 데이터 정리

## 경쟁사 비교

| | 우리 | SK AX/솔트룩스 | 한국GPT협회 | SaaS (올거나이즈 등) |
|---|---|---|---|---|
| **뭘 해주나** | PC에 깔아줌 + 운영 | 컨설팅 + 구축 | 교육만 | 클라우드 구독 |
| **도입 비용** | 150~200만원 | 수천만원~억 | 교육비만 | 월 수십~수백만원 |
| **도입 기간** | 3~5일 | 수주~수개월 | 교육 후 알아서 | 즉시 (but 맞춤화 X) |
| **맞춤화** | 기업 양식 학습 | 풀커스텀 | 없음 | 제한적 |
| **사용법** | 텔레그램 채팅 | 전용 시스템 | 직접 사용 | 웹 UI |
| **유지보수** | 원격 (월 30~50만) | 별도 계약 | 없음 | 구독료에 포함 |
| **대상** | 중소기업 | 대기업/중견 | 전체 | 중견 이상 |

## 도입 프로세스

```
1단계: 현장 분석 (반나절)
   └── 업무 파악, 데이터 흐름 확인, 양식 수집

2단계: PC 세팅 (2~3시간)
   └── WSL2 설치, Claude Code, 텔레그램 봇 연결

3단계: 맞춤 설정 (3~5시간)
   └── 기업 양식 학습, 분석 템플릿, 크론잡 설정

4단계: 교육 (1~2시간)
   └── "텔레그램으로 이렇게 보내면 됩니다" + 매뉴얼

5단계: 운영 & 유지보수 (월간)
   └── 원격 모니터링, 템플릿 수정, 최적화
```

## 기술 스택

| 분류 | 기술 |
|------|------|
| OS | Windows 10/11 + WSL2 (Ubuntu) |
| AI 엔진 | Claude Code (Anthropic) |
| 커뮤니케이션 | Telegram Bot API |
| 데이터 분석 | Python (pandas, matplotlib, numpy) |
| 문서 생성 | python-docx (워드), openpyxl (엑셀), pyhwp (한글) |
| 음성→텍스트 | Whisper (OpenAI STT) |
| 자동화 | Cron, inotifywait (파일 감시) |
| DB 연동 | MySQL, PostgreSQL, MSSQL, SQLite |
| 원격 관리 | SSH, RDP |

### 세팅 요구사항
- Windows 10/11 PC (기존 PC 활용 가능)
- RAM 8GB 이상 (16GB 권장)
- SSD 256GB 이상
- 유선 인터넷

## 보안

- Anthropic 정책: 입력 데이터를 AI 학습에 사용하지 않음
- 데이터는 분석 시에만 전송, 서버에 저장하지 않음
- 텔레그램 접근 권한 관리 (allowlist — 승인된 직원만)
- VPN/방화벽 내 운영 가능
- 민감 데이터는 로컬 전용 처리 옵션

## 영업 전략

### 타겟
- 직원 5~50명 규모 중소기업
- 엑셀/한글 서류 많이 쓰는 업종 (건설, 제조, 도매)
- "AI 쓰고 싶은데 뭘 어떻게 해야할지 모르는" 사장님

### 영업 포인트
1. **데모**: "사장님 회사 양식 하나 주세요, 지금 바로 보여드릴게요"
2. **실적**: "저도 이렇게 쓰고 있습니다" (본인 사용 사례)
3. **가격**: "컨설팅업체 가면 수천만원, 여기는 200만원이면 끝"
4. **간편함**: "직원분들은 텔레그램으로 보내기만 하면 됩니다"

### 확장
- 1차: 주변 사장님 네트워크 → 시연 → 계약
- 2차: 업종별 사례 축적 → 블로그/유튜브 마케팅
- 3차: 세팅 자동화 → 규모 확장
- 4차: 다른 세팅 기사 양성 (프랜차이즈)

# BIZ AI Assistant

## "PC 한 대 깔아드립니다" — 기업용 AI 비서 세팅 사업

> 직원이 재료만 던지면, AI가 완성된 문서를 만들어서 돌려주는 구조

---

## 한줄 요약

Windows PC 한 대에 AI를 세팅해서 기업에 납품.
엑셀, 한글, 워드, 녹음파일 — 뭘 줘도 AI가 알아서 분석하고 문서를 생성한다.

---

## 이 사업이 왜 되나?

### 시장 상황 (2026년)
- 국내 기업 85%가 생성형 AI 도입 원함
- 근데 "어떻게 해야할지 모름"이 가장 큰 장벽
- 대형 컨설팅(SK AX, 솔트룩스): 수천만원~억 단위 → 중소기업 접근 불가
- 교육업체(한국GPT협회 등): 가르치기만 함, 직접 세팅 안 해줌
- 정부 바우처: 신청 복잡, 6개월 기다림

### 빈틈
**"저렴하게 직접 깔아주는" 서비스가 없다.**

### 우리 포지션: "동네 IT 형"
- 150~200만원이면 끝 (대형 대비 1/10)
- 3~5일이면 완료 (대형은 수개월)
- 교육이 아니라 직접 세팅 + 운영
- 텔레그램 채팅으로 바로 쓸 수 있는 실용적 결과물

## 핵심 기능: "양식 + 데이터 → 새 문서 자동 생성"

### 이런 게 된다

**회의록 자동 작성**
```
직원이 텔레그램으로:
"회의록 작성해줘
 - 녹음파일: (음성파일 전송)
 - 장소: 본사 3층 회의실
 - 시간: 4/16 오후 2시
 - 참석자: 김과장, 이대리, 박부장"

→ AI가:
1. 음성 → 텍스트 변환 (STT)
2. 안건/결정사항/TODO 자동 분류
3. 기업 양식에 맞춰 회의록 작성
4. 한글(.hwp) 또는 워드(.docx) 파일 생성
5. 텔레그램으로 완성 파일 회신
```

**보고서 자동 생성**
```
공유폴더에 매출 엑셀 넣으면
→ AI가 읽고 분석
→ 월간 매출 보고서 자동 생성 (차트 포함)
→ 텔레그램으로 발송
```

**서식 학습**
```
처음에: "우리 회의록 양식 이거야" (파일 전달)
이후: 그 양식 그대로 새 파일 찍어냄
매번 같은 포맷으로 생성
```

### 생성 가능한 파일 형식
| 형식 | 지원 | 용도 |
|------|------|------|
| 엑셀 (.xlsx) | O | 표, 차트, 수식 포함 |
| 워드 (.docx) | O | 보고서, 회의록 |
| 한글 (.hwp) | O | 관공서/건설 서류 |
| PDF | O | 최종 배포용 |
| CSV/JSON | O | 데이터 내보내기 |
| 이미지/차트 | O | 시각화 결과물 |

### 분석 가능한 입력
| 입력 | 처리 |
|------|------|
| 엑셀/CSV 파일 | 데이터 분석, 이상치 탐지, 트렌드 |
| 음성 녹음 | STT → 텍스트 → 회의록 |
| 한글/워드 문서 | 내용 분석, 요약, 분류 |
| 이미지 | OCR → 텍스트 추출 |
| DB 데이터 | 자연어 쿼리 ("이번 달 매출 뽑아줘") |

## 제공 서비스

### 1. 데이터 분석
- Excel/CSV/JSON 파일 자동 분석
- 이상치 탐지, 패턴 분석, 트렌드 파악
- 시각화 차트 자동 생성
- 업종별 맞춤 분석 (매출/재고/인사/공정)

### 2. 문서 자동 생성
- 보고서 자동 작성 (일일/주간/월간)
- 회의록 (녹음 → 문서)
- 계약서/견적서 초안
- 기업 양식 학습 → 동일 포맷 반복 생성

### 3. 파일 분류 & 정리
- 공유폴더 파일 자동 분류
- 문서 내용 기반 태깅
- 중복 파일 탐지
- 폴더 구조 자동 정리

### 4. DB 연동 분석
- 기업 DB 직접 연결 (MySQL/PostgreSQL/MSSQL)
- 자연어로 쿼리 ("거래처별 미수금 뽑아줘")
- 정기 모니터링 & 이상 시 알림

### 5. 정기 자동화
- 매일 아침 일일 리포트 자동 생성
- 매주 월요일 주간 분석 보고
- 새 파일 감지 → 자동 처리
- 텔레그램으로 결과 발송

## 시스템 구조

```
Windows PC (기업 현장 — 기존 PC 활용 가능)
│
├── WSL2 (Ubuntu — Windows 안의 리눅스)
│   ├── Claude Code (AI 엔진)
│   ├── 텔레그램 봇 (직원↔AI 소통 채널)
│   ├── 크론잡 (자동 스케줄)
│   │   ├── 매일 09:00 — 일일 리포트
│   │   ├── 매주 월 — 주간 분석
│   │   └── 파일 감시 — 새 파일 자동 처리
│   └── 분석 도구 (Python, Node.js)
│
├── 공유 폴더 (사내 네트워크)
│   ├── /입고/ — 직원이 파일 넣는 곳
│   ├── /결과/ — AI가 처리 결과 저장
│   └── /보고서/ — 자동 생성 보고서
│
├── DB 연결 (선택)
│   └── 기존 ERP/회계 시스템 연동
│
└── 원격 관리 (RDP/SSH)
    └── 외부에서 유지보수
```

### 사용 흐름
```
직원 → 텔레그램 메시지 or 공유폴더에 파일 드롭
  ↓
AI가 분석/처리/문서 생성
  ↓
결과를 텔레그램으로 회신 or 공유폴더에 저장
```

## 보안

- Anthropic 정책: 입력 데이터를 AI 학습에 사용하지 않음
- 데이터는 분석 시에만 전송, 서버에 저장하지 않음
- 텔레그램 접근 권한 관리 (allowlist — 승인된 직원만)
- VPN/방화벽 내 운영 가능
- 민감 데이터는 로컬 전용 처리 옵션

## 영업 전략

### 타겟
- 직원 5~50명 규모 중소기업
- 엑셀/한글 서류 많이 쓰는 업종 (건설, 제조, 도매)
- "AI 쓰고 싶은데 뭘 어떻게 해야할지 모르는" 사장님

### 영업 포인트
1. **데모**: "사장님 회사 양식 하나 주세요, 지금 바로 보여드릴게요"
2. **실적**: "저도 이렇게 쓰고 있습니다" (본인 사용 사례)
3. **가격**: "컨설팅업체 가면 수천만원, 여기는 200만원이면 끝"
4. **간편함**: "직원분들은 텔레그램으로 보내기만 하면 됩니다"

### 확장
- 1차: 주변 사장님 네트워크 → 시연 → 계약
- 2차: 업종별 사례 축적 → 블로그/유튜브 마케팅
- 3차: 세팅 자동화 → 규모 확장
- 4차: 다른 세팅 기사 양성 (프랜차이즈)

## 세팅기 (Configuration Tool)

### 개념
기업별 AI 세팅을 표준화하고 정확도를 높이기 위한 설정 도구.
"AI가 90% 자동 세팅 → 관리자가 10% 검토/수정" 구조.

### 작동 방식
```
1. 기업 자료 투입 (조직도, 명함, 기존 양식, 문서 등)
   ↓
2. AI가 자동 분석 → 세팅값 자동 생성
   - 인물사전 추출 (이름/직책/부서)
   - 고유명사 추출 (회사명, 프로젝트명)
   - 양식 구조 분석
   - 업종 판단 → 프리셋 자동 적용
   ↓
3. 세팅기 화면에 결과가 채워져 있음
   ↓
4. 관리자가 검토 후 틀린 것만 수정
   ↓
5. 저장 → config.json + CLAUDE.md 자동 생성
   ↓
6. 모든 Claude 세션이 이 설정을 따라감
```

### 세팅기 기능

| 기능 | 설명 |
|------|------|
| **기능 토글** | 회의록/보고서/일보/STT/DB연동 등 ON/OFF 스위치 |
| **양식 관리** | 양식 파일 업로드 + 필드 매핑 (드래그앤드롭) |
| **인물사전** | 이름 + 직책 + 발음 변형 등록 (STT 인식 보정) |
| **고유명사 사전** | 회사명, 프로젝트명, 약어 매핑 |
| **프로젝트/거래 메모** | 배경 정보 등록 (맥락 파악용) |
| **요약 레벨** | 상세/보통/간략 + 필수 포함 키워드 |
| **검수 모드** | ON 시 초안 확인 후 최종 발송 |
| **STT 교정 규칙** | 자주 틀리는 단어 매핑 |
| **출력 설정** | docx/xlsx/pdf 선택, 로고/회사명 자동 삽입 |
| **파일 경로** | 폴더 찾기 버튼 or 현재 위치 자동 설정 |

### 생성되는 파일
```
프로젝트 폴더/
├── CLAUDE.md          ← AI 행동 규칙 (모든 세션이 참조)
├── config.json        ← 기업 설정 데이터
├── dictionary/
│   ├── people.json    ← 인물사전
│   ├── terms.json     ← 고유명사/약어
│   └── corrections.json ← STT 교정 규칙
└── templates/
    ├── 회의록.docx
    ├── 일일보고.xlsx
    └── ...
```

### 핵심 원칙
> **절대 내용을 지어내지 않는다.**
> STT 결과는 들린 그대로, 데이터는 있는 그대로.
> 안 들리면 "불명확" 표시, 정보 없으면 비워둔다.
> 한 번이라도 지어낸 내용이 들어가면 서비스 신뢰가 무너진다.

## 왜 SaaS(API)보다 PC 설치가 나은가

| | PC 설치 (우리) | SaaS (API 호출) |
|---|---|---|
| **컨텍스트** | 이전 대화/설정 기억 | 매번 새로 시작 |
| **파일 접근** | 로컬 파일 직접 읽기/쓰기 | 업로드/다운로드 필요 |
| **도구 조합** | Whisper+pandas+docx 자유 조합 | API 하나로 제한 |
| **정확도** | 세팅기로 기업맞춤 튜닝 | 범용 프롬프트 |
| **피드백** | 실시간 수정/반영 | 재호출 필요 |
| **비용** | 월 3~7만원 (API) | 월 수십만원 (구독) |
